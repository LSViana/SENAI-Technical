Sistema de alteração de dados (por exemplo: senha) através de (links com) TOKENS recebidos por e-mail, onde poderia haver uma $FUNCTION para inserir um novo token na tabela de tokens válidos e retorná-lo para geração do link para o usuário, e uma $VIEW que, através de um JOIN, permite ao Administrador do sistema visualizar quais TOKENS estão associados a quais usuários, por fim, podemos uma utilizar uma $PROCEDURE que limpa os TOKENS antigos o suficiente do banco de dados. (Poderíamos também utilizar uma $FUNCTION para que, quando o usuário acesse o link/insira o TOKEN, validá-lo e removê-lo da base de dados e retornar se houve sucesso.)
